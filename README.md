# Hermes Content Agent

Автономный контент-агент, который живёт у тебя в Obsidian и пишет посты твоим голосом.

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

Не сервис, не подписка, не «вход через нашу платформу». Ты ставишь его у себя на компе, на своих ключах, и он бесплатный. Этот репозиторий — полная инструкция и готовый скелет, который ты разворачиваешь за один вечер.

Я каждый день гоняю на этом свой контент: идея → агент пишет 3 варианта твоим голосом → ты выбираешь и постишь. Внизу расскажу, кто я и зачем отдаю это даром.

---

## Что у тебя будет в конце

Пройдёшь по шагам — получишь:

- рабочую систему, которая превращает сырую мысль или голосовую заметку в готовый пост;
- посты, которые звучат как **ты**, а не как причёсанный текст из методички;
- структуру в Obsidian, где видно: какие темы в очереди, что в работе, что уже опубликовано;
- понятный путь «идея → черновик → апрув → пост» примерно на 15–20 минут вместо часа.

Всё крутится локально. Никто не держит твой контент у себя на сервере.

## Кому это надо

- ведёшь личный бренд или продукт и устал постить руками;
- хочешь системность, а не «когда придёт вдохновение»;
- готов один раз потратить вечер на сетап.

## Кому не надо

- ищешь кнопку «сделай за меня без настройки» — это не сюда, тут надо один раз собрать;
- не готов описать свой голос словами (без этого агент пишет слоп, см. шаг про `voice.md`).

---

## Из чего собрано

Три части, каждая бесплатна на старте:

| Часть | Что делает | Зачем |
|---|---|---|
| **Obsidian** | хранилище: темы, черновики, правила голоса, логи | всё лежит обычными markdown-файлами, ничего не залочено |
| **Hermes** | агент в терминале: читает твой vault, ходит в модели, пишет черновики | это «мозг», который связывает всё вместе |
| **OpenRouter** | один ключ к сотне моделей (Claude, GPT, DeepSeek и др.) | не надо регать аккаунт под каждую модель |

## Что понадобится

- компьютер (Mac или Windows);
- почта (Gmail подойдёт);
- ~30–40 минут на сетап;
- 0 рублей. Кредитка не нужна — у OpenRouter есть бесплатные модели.

## Установка

### Шаг 1. Node.js

Это то, на чём запускается Hermes.

Проверь, есть ли уже:

```
node --version
```

Видишь `v18` или выше — пропускай шаг. Видишь ошибку — ставь с [nodejs.org](https://nodejs.org) (версия LTS) или через Homebrew: `brew install node`.

### Шаг 2. Obsidian

Хранилище идей, черновиков и логов. Это просто заметки в markdown.

Скачай с [obsidian.md](https://obsidian.md), поставь, открой. Папку-vault создадим на шаге 5 — пока просто установи.

### Шаг 3. OpenRouter

Мост к моделям. Один ключ — доступ ко всем.

1. Заходишь на [openrouter.ai](https://openrouter.ai), регаешься (Google проще всего).
2. Кредитка не нужна.
3. `API Keys` → `Create Key` → назови `hermes` → скопируй.
4. Сохрани ключ, он вида `sk-or-...`.

Бесплатные модели для старта: `deepseek/deepseek-v4-flash:free`, `google/gemma-4-31b-it:free`.
Кинешь позже $5–10 — откроются тяжёлые для драфтинга: `anthropic/claude-opus-4.8`, `deepseek/deepseek-v4-flash`.

### Шаг 4. Hermes

```
npm install -g hermes-agent
hermes --version
```

Видишь номер версии — поставилось. Если на Mac ругается на права — `sudo npm install -g hermes-agent`. На Windows — запусти терминал от администратора.

### Шаг 5. Конфиг

Создай папку и файл `~/.hermes/config.yaml`:

```yaml
model:
  default: deepseek/deepseek-v4-flash:free
  provider: openrouter
  base_url: https://openrouter.ai/api/v1
terminal:
  backend: local
  timeout: 180
```

Пропиши ключ в окружение (Mac, добавь в `~/.zshrc`):

```
export OPENROUTER_API_KEY="sk-or-...твой ключ..."
```

Перезапусти терминал, проверь: набери `hermes`, увидишь приглашение — всё связано.

## Структура vault

Вот скелет, который ты кладёшь в свой Obsidian. Лежит в этом репозитории в папке `vault-template/`.

```
vault/
├── _shared/
│   ├── profile.md          ← кто ты, продукт, аудитория, цели
│   ├── voice.md            ← твой голос (самый важный файл)
│   ├── avoid-slop.md       ← чего агент НЕ делает
│   └── hooks.md            ← формулы первых строк
├── content/
│   ├── topics-pipeline.md  ← очередь тем: что пишем, на какой платформе
│   ├── funnel-map.md       ← как платформы связаны в воронку
│   └── inbox.md            ← сюда кидаешь сырые идеи, одной строкой
├── research/              ← транскрипты постов/видео, которые зашли — источник идей
├── agents/
│   ├── strategist/agent.md ← наполняет очередь темами (не пишет посты)
│   └── writer/agent.md     ← пишет посты по очереди
└── runs/
    ├── _template/          ← болванка папки одного поста
    ├── active/             ← посты в работе
    └── archive/            ← опубликованные
```

Логика простая: **стратег** держит очередь тем (`topics-pipeline.md`), **писатель** берёт тему и пишет 3 варианта. Каждый пост — отдельная папка в `runs/`, чтобы видеть путь от идеи до публикации.

---

## Как развернуть структуру: два способа

### Способ A — через промт (быстрее)

Открой `setup-prompt.md` из этого репозитория. Скопируй его целиком, вставь Hermes-агенту. Он создаст всю структуру у тебя в vault сам, и по ходу спросит про твой продукт, аудиторию и голос. Это и есть демо системы: агента ты настраиваешь, разговаривая с агентом.

### Способ B — вручную

Склонируй репозиторий и скопируй папку `vault-template/` в свой Obsidian:

```
git clone https://github.com/alexey-muzyka-1/hermes-content-agent.git
```

Дальше открой папку как vault в Obsidian и заполни файлы из `_shared/` под себя.

## Самый важный шаг: опиши свой голос

Открой `_shared/voice.md` и заполни его честно. Без этого агент пишет гладкий, умный, мёртвый текст — слоп.

Что туда писать:

- как ты разговариваешь (коротко/длинно, грубо/мягко, с матом/без);
- 5–10 твоих живых фраз, которые агент должен сохранять, а не «причёсывать»;
- темы, на которые ты говоришь искренне;
- чего ты **никогда** не пишешь (пафос, «представь», POV, инфоцыганщина).

Правило, которое экономит тебе нервы: **если текст звучит как пост любого другого человека — это слоп, переписать.**

## Откуда агент берёт идеи

Два источника, можно оба:

1. **Из головы.** Кидаешь мысль в `content/inbox.md` одной строкой или диктуешь голосом.
2. **Из ресерча.** Складываешь в папку `research/` транскрипты постов и видео, которые тебе зашли — то, в чьих темах и стиле хочешь работать. Агент читает их и достаёт оттуда идеи и разборы, а не выдумывает из воздуха.

Второй источник сильнее: чем больше насмотренного лежит в `research/`, тем точнее темы.

## Как работает один пост

1. **Идея.** Из `content/inbox.md` (своя мысль) или из `research/` (разбор того, что зашло).
2. **Очередь.** Стратег раскидывает идеи по платформам и этапам воронки в `topics-pipeline.md`.
3. **Папка поста.** Писатель копирует `runs/_template/` в `runs/active/ГГГГ-ММ-ДД-слаг/`.
4. **3 варианта.** Агент пишет три версии с разными хуками в `draft-package.md`.
5. **Анти-слоп.** Прогоняет через `avoid-slop.md`: вернуть живую сцену, убрать методичку, сохранить твои слова.
6. **Ты выбираешь.** Читаешь, берёшь лучший, правишь под себя.
7. **Постишь сам.** Никакого автопостинга. Ты решаешь, что выходит в мир.
8. **Лог.** Папка едет в `runs/archive/`, результат записывается.

## Сколько занимает

- сетап (шаги 1–5) — один вечер, ~40 минут;
- заполнить `voice.md` и `profile.md` — за один присест;
- первый пост — в тот же день.

Голос ловится не с первого раза. Первые 3–4 поста — калибровка `voice.md`, дальше быстрее.

## Частые затыки

- **`hermes: command not found`** — закрой и заново открой терминал.
- **Агент пишет гладко и мёртво** — `voice.md` пустой или общий. Добавь свои живые фразы.
- **Модель отвечает ерунду на бесплатном тарифе** — для драфтинга подключи платную модель только на шаге написания, остальное оставь на бесплатной.

## Как усилить агента

- **Подключи [Viralmaxing MCP](https://viralmaxing.com/docs/assistant/mcp).** Трендвотчинг и аналитика рилсов прямо в работу: забираешь самые трендовые рилсы и переписываешь под них сценарий, не выходя из vault.
- **Научи его искать.** Любишь возиться с агентами — дай ему поиск (web, Reddit, X) и свои инструменты. Больше источников — меньше выдумок.
